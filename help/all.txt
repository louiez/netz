LAN IPs
After the Page loads, the NETz server will Ping the LAN IP and Gateway and
display
[../online.gif] if Online
[../down.gif] if down
The page will request the NETz server check this every 15 seconds and change
the image if online status changes .
Buttons
There are 3 buttons [P][T][S][W] for Ping, Telnet,SSH and Web.
[P] The ping button will open the ping window and ping the IP in this field.

[T] Telnet button will attempt to open a Telnet session with the default telnet
client on your system. you will need to alter the registry to use the correct
application for telnet the registry key is
HKEY_CLASSES_ROOT\telnet\shell\open\command if you are not sure how to do this
get with your PC support

[S] the SSH button will open the application used with SSH URL handler from a
browser normally windows dosen't have a URL handler for SSH and you will need
to add one just like the Telnet URL handler located at HKEY_CLASSES_ROOT\

[W] Web button will attempt to web into the site ie: ( http://FIELD-IP ) from
the web browser

Monitor Cycle interval
This is the amount of Time the server will wait before starting another Ping
cycle.
NETz will complete all monitor pings for all sites within this cycle...

NETz Pings each site 3 times per cycle. So a monitor cycle set for 10 minutes
will ping each site 3 times every 10 min.

NETz Plugins
Creating NETz plugin
The NETz plugin was created because most times "one size fits all" just don't
work.
So with plugins you can add function, links and messages without breaking the
NETz core files.
This plugin system lets you upgrade NETz and just save and restore the Plugins
directory.

Each plugin can be controled as to what level of user has access to it.
if a plugin is set to user level 2, anybody 2 or higher will activate that
plugin.

NETz Plugin can be a link to a file, page or site
Plugins can fun from
Main page
Ops page
Support Page
Site check script
See list below for exact locations

Netz has predetermined locations it will check for plugin inclusion.
when it gets to these locations it checks
1. what plugins have been enabled
2. if this plugin was set to run at this location
3. If the current user has the correct level to run this plugin
4. it looks at the plugings ini file to see how this plugin is run



Below are the location tags within a plugin.ini file that can be used
plugin_info
support_footer - this will run at the very bottom of the support page just
befor the NETz footer note
support_middle_1 - this is in the table cell on row 7 last column
support_middle_2 - this is in the table cell on row 10 last column
support_left
support_load
main_admin
main-load
main_footer
main_info
main_tools
ops_top_right
ops_middle_1
ops_middle_2
ops_load
ops_footer
ops_bottom_right
site_check_start
site_check_end

Delete the location section not needed


Netz searches the plugins for directories and assumes any directory is a
plugin.
then it reads the main plugin.ini file to see what plugins are enabled.
It then will look in each directory for a plugin.ini file and read its
contents.

your included file can access the javascript variable 'site' to get the SITE_ID
or
if you have a link added it will be passed the GET site  ie: http://
yoururl.php?site=

SMTP Auth
Many providers now block outbound port 25 (email) traffic.
in order to send mail you must relay off your providers email server.
then this relay started to become an issue so now most will require you
to not only relay off their server but logon first with a valid account using
SMTPAUTH.
some providers even go one step further, and require a valid "authorized" from
email address.

Now with that rant done.
If you want NETz to send alert emails and you have a provider with SMTPAUTH
requirements.
Netz provides a place to enter a valid User, password and from address.
