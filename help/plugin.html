NETz Plugins

Creating NETz plugin
The NETz plugin was created because most times "one size fits all" just don't work.
So with plugins you can add function, links and messages without breaking the NETz core files.
This plugin system lets you upgrade NETz and just save and restore the Plugins directory.

Each plugin can be controled as to what level of user has access to it.
if a plugin is set to user level 2, anybody 2 or higher will activate that plugin.

NETz Plugin can be a link to a file, page or site
Plugins can fun from
Main page
Ops page
Support Page
Site check script
See list below for exact locations

Netz has predetermined locations it will check for plugin inclusion.
when it gets to these locations in NETz code it checks 
1. what plugins have been enabled for this section of NETz code
2. if this plugin was set to run at this location
3. If the current user has the correct level to run this plugin
4. it looks at the plugin's ini file to see how this plugin is run

-----------------------------------------------------------------------
there needs to be a plugin.ini within each plugin directory
the main section of this file is
[plugin_info]
description :	This is the description displayed in the NETz config Page plugin tab
version :	Version of the Plugin... if any
name :		The name displayed in the NETz config Page plugin tab
web_site :	Plugin Developers Web site
email :		Email of developer or maintaing of plugin
help :		URL to any Help file

-----------------------------------------------------------------------
Below are the location tags in NETz code that can be used - set in the plugin.ini file

[logon_complete]
[support_footer] - bottom of the support page just before the NETz footer note
[support_middle_1] - table cell on row 7 last column
[support_middle_2] - table cell on row 10 last column
[support_left]
[support_load]
[main_admin]
[main-load]
[main_footer]
[main_info]
[main_tools]
[ops_monitor_panel]
[ops_top_right]
[ops_middle_1]
[ops_middle_2]
[ops_load]
[ops_footer] - bottom of the ops page just before the NETz footer note
[ops_bottom_right]
[site_check_start]
[site_check_end]

Delete the location section not needed to make the ini file look cleaner

---------------------------------------------------------------------------
Within each location section there are the following options
link :		the link to the page. 
link_path :	If above link is an absolute URL, like a link to another site set link_path to "absolute"
		If set to "relative" or left blank it is assumed relitive to the plugin Directory
link_name :	Link displayed name
link_target :	Target page - some of these act different on different Browsers
		_blank
		_self
		_parent
		_top
		special for plugins are
			_new          creats a new window
			_message      creates a new window and not allow a duplicate
			   NOTE: The message window will stop duplicates 
				 only if the parent page doesn't navigate to another page
file :		If you want to just inject some code at the section location this will be the include file

********** Below is for link_target's "_new" and "_message" only *********
width :		This is the window Width
height : 	This is the window Height
resizable : 	Set to "yes" or "No" to allow resizing of window
scrollbars :	Set to "yes" or "No" to allow scroll bars on window
status : 	Set to "yes" or "No" to allow Status at bottom of Window

-----------------------------------------------------------------------------

Netz searches the plugins for directories and assumes any directory is a plugin.
then it reads the main plugins.ini file to see what plugins are enabled.
It then will look in each directory for a plugin.ini file and read its contents.

your included file can access the javascript variable 'site' to get the SITE_ID
or
if you have a link added it will be passed the GET site  ie: http://yoururl.php?site=
